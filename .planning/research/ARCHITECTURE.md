## 二、LangChain组件架构

### 2.1 LCEL核心模式

LangChain Expression Language (LCEL) 是LangChain的声明式组合方式，推荐用于所有新链：

```
┌─────────────────────────────────────────────────────────────────┐
│                    LCEL 组合模式示意                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  基础序列 (RunnableSequence)                                    │
│  ────────────────────────────                                  │
│  prompt | llm | output_parser                                   │
│                                                                 │
│  检索链 (Retrieval Chain)                                       │
│  ─────────────────────────                                      │
│  retriever | format_docs | llm | output_parser                  │
│                                                                 │
│  并行执行 (RunnableParallel)                                    │
│  ─────────────────────────                                      │
│  RunnableParallel({                                             │
│    "query": query,                                              │
│    "context": retriever,                                        │
│    "history": memory                                            │
│  })                                                             │
│                                                                 │
│  条件分支 (RunnableBranch)                                      │
│  ────────────────────                                           │
│  RunnableBranch(                                                │
│    (is_simple, simple_chain),                                   │
│    (is_complex, complex_chain),                                 │
│    default_chain                                                │
│  )                                                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 Chain vs Agent 选择

| 类型 | 适用场景 | 学术RAG中的应用 |
|------|----------|-----------------|
| **Chain** | 固定工作流、确定性流程 | PDF解析、检索-生成基础链 |
| **Agent** | 多步骤决策、自适应任务 | 复杂查询分解、多源检索协调 |

### 3.3 数据流转关键点

| 阶段 | 输入 | 输出 | 关键技术 |
|------|------|------|----------|
| **文档解析** | PDF/arXiv | 结构化文本块 | PyPDF, Unstructured, PaddleOCR |
| **分块** | 原始文本 | 重叠语义块 | Recursive splitter, Semantic splitter |
| **向量化** | 文本块 | 768维向量 | Sentence-BERT, BAAI/bge |
| **检索** | 查询向量 | Top-K相关块 | Cosine similarity, HNSW索引 |
| **重排序** | 初检结果 | 精排结果 | Cross-encoder, Cohere Rerank |
| **生成** | 上下文+问题 | 带引用的答案 | Minimax-m2.1 API |

## 六、引用溯源数据结构

### 6.1 元数据模型

```python
# 文档块元数据模型
class DocumentChunk(BaseModel):
    chunk_id: str                    # 唯一标识 UUID
    document_id: str                 # 所属论文ID
    document_title: str              # 论文标题
    chunk_index: int                 # 在文档中的顺序
    page_start: int                  # 起始页码
    page_end: int                    # 结束页码
    content: str                     # 原始内容
    content_type: ContentType        # TEXT/IMAGE/TABLE/FORMULA
    embedding: List[float]           # 向量表示
    metadata: Dict[str, Any]         # 额外元数据
    created_at: datetime             # 入库时间
    source_reference: SourceRef      # 原始来源引用

# 引用溯源结构
class SourceRef(BaseModel):
    source_type: str                 # "pdf" / "arxiv" / "url"
    source_id: str                   # 文件哈希或DOI
    original_path: Path              # 原始文件路径
    page_number: int                 # 页码 (1-based)
    line_range: Tuple[int, int]      # 行号范围
    bbox: Optional[Tuple]            # 图像区域坐标
    confidence: float                # OCR置信度

# 答案引用结构
class AnswerCitation(BaseModel):
    citation_id: str                 # 引用编号 [1]
    chunk_ids: List[str]             # 关联的chunk ID列表
    source_refs: List[SourceRef]     # 原始来源
    quoted_text: str                 # 引用的原文片段
    relevance_score: float           # 相关性评分
    position: CitationPosition       # 在答案中的位置
```

### 6.2 引用追踪流程

```

    ▼

                                        │

                                        │
                                        ▼
                               ┌─────────────────┐
                               │  引用索引构建   │
                               │  • 块ID → 位置  │
                               │  • 原文 → 引用  │

                                        │
                                        ▼
                               带引用的结构化答案
                               [{"1": {"text": "...", "page": 5}}]
```

### 8.2 插件注册机制

```python
# 插件注册示例
class PluginRegistry:
    _loaders: Dict[str, Type[BaseLoader]] = {}
    _embeddings: Dict[str, Type[BaseEmbedding]] = {}
    _llms: Dict[str, Type[BaseLLM]] = {}
    _ retrievers: Dict[str, Type[BaseRetriever]] = {}

    @classmethod
    def register_loader(cls, name: str, loader_cls: Type[BaseLoader]):
        cls._loaders[name] = loader_cls

    @classmethod
    def get_loader(cls, name: str) -> Type[BaseLoader]:
        return cls._loaders[name]

# 使用
PluginRegistry.register_loader("arxiv", ArxivLoader)
PluginRegistry.register_loader("pdf", PDFLoader)
```

### 9.1 优化策略矩阵

| 优化维度 | 策略 | 实施阶段 | 复杂度 |

| **并发能力** | 异步处理 + 连接池 | Phase 9 | 高 |
| **缓存** | Redis缓存查询结果 | Phase 6 | 低 |

## 十、架构决策记录

| 决策点 | 选择 | 权衡考量 |
|--------|------|----------|
| 核心框架 | LangChain LCEL | 生态丰富、组件标准化、社区活跃 |
| 向量数据库 | Chroma (开发) → Pinecone (生产) | 本地开发低成本、云端生产高可用 |
| 嵌入模型 | BGE (中英) / OpenAI | 兼顾中英双语、开源免费 |
| LLM | minimax-m2.1 | 免费API、适合学术场景 |
| 检索策略 | Hybrid + Re-rank | 平衡召回与精度 |
| 部署方式 | Docker + 云服务器 | 面试展示需求 |
