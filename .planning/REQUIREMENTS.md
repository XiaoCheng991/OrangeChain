## 文档解析类需求 (DOC)

### REQ-DOC-001: PDF文件上传
**优先级**: P0 | **复杂度**: Low | **预估工时**: 2-3h

**描述**: 用户可以上传PDF格式的学术论文文件。

**验收标准**:
- [ ] 支持拖拽上传和文件选择器
- [ ] 支持单个文件最大 50MB
- [ ] 上传进度可视化
- [ ] 支持批量上传（最多10个文件）
- [ ] 上传后自动解析文档基本信息（页数、大小）

**技术实现**:
- 前端: react-dropzone / vue-upload
- 后端: PyMuPDF ( Fitz )

---

### REQ-DOC-002: arXiv链接抓取
**优先级**: P0 | **复杂度**: Medium | **预估工时**: 3-4h

**描述**: 用户可以输入arXiv论文链接，系统自动下载并解析论文。

**验收标准**:
- [ ] 支持标准arXiv链接格式 (https://arxiv.org/abs/xxx)
- [ ] 自动提取arXiv ID并下载PDF
- [ ] 解析论文元数据（标题、作者、摘要、分类）
- [ ] 处理arxiv.org临时不可用的情况（重试机制）

**技术实现**:
- arXiv API: http://export.arxiv.org/api/query
- 下载: requests 库 + 重试策略

---

### REQ-DOC-003: 纯文本提取与分块
**优先级**: P0 | **复杂度**: Medium | **预估工时**: 3-4h

**描述**: 从PDF中提取文本并按语义分块存储。

**验收标准**:
- [ ] 提取全文文本（忽略页眉页脚）
- [ ] 保留段落结构（不破坏段落完整性）
- [ ] 基于页面+段落的逻辑分块
- [ ] 每个chunk保留元数据（页码、所属章节）

**技术实现**:
- PyMuPDF: `page.get_text("blocks")` + 段落重组
- 分块策略: 页面边界 + 段落完整性

**优先级**: P1 | **复杂度**: High | **预估工时**: 5-6h

**描述**: 识别并提取论文中的表格内容。

**验收标准**:

## 检索类需求 (RET)

### REQ-RET-001: 向量检索
**优先级**: P0 | **复杂度**: Medium | **预估工时**: 4-5h

**描述**: 使用向量数据库实现语义相似度检索。

**验收标准**:
- [ ] 文档分块后生成向量并存入数据库
- [ ] 支持语义查询（问题式检索）
- [ ] 返回top-k相关片段（默认k=5）
- [ ] 检索延迟 < 500ms

**技术实现**:
- 向量库: ChromaDB (本地/演示友好)
- Embedding: BGE-M3 (多语言、支持中英)
- LCEL: `RetrievalChain` + `RunnablePassthrough`

---

### REQ-RET-002: 关键词检索 (BM25)
**优先级**: P0 | **复杂度**: Low | **预估工时**: 2-3h

**描述**: 实现BM25算法进行精确关键词匹配。

**验收标准**:
- [ ] 对专业术语实现精确匹配
- [ ] 独立于向量检索运行
- [ ] 返回相关性评分

**技术实现**:
- rank-bm25 库
- 独立索引，适配混合检索

---

### REQ-RET-003: 混合检索 (RRF)
**优先级**: P1 | **复杂度**: High | **预估工时**: 6-8h

**描述**: 融合向量检索和BM25结果，提升检索质量。

**验收标准**:
- [ ] 同时执行向量检索和BM25检索
- [ ] 使用RRF(倒数排名融合)算法合并结果
- [ ] 向量:BM25权重可配置（默认6:4）
- [ ] 最终结果排序优于单一检索

**技术实现**:
- RRF公式: 1/(rank_V + k) + 1/(rank_B + k)
- k=60 避免除零

---

### REQ-RET-004: 语义重排序
**优先级**: P1 | **复杂度**: Medium | **预估工时**: 4-5h

**描述**: 对初步检索结果进行二次精排。

**验收标准**:
- [ ] 对混合检索结果（top-20）进行重排序
- [ ] 使用Cross-encoder重新计算相关性
- [ ] 返回更精确的top-10结果

**技术实现**:
- BGE-Reranker (bge-reranker-base)
- HuggingFace Embeddings 加载
